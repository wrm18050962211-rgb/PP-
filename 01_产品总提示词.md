# 01_产品总提示词

请帮我设计一个陪伴型拍照搭子平台 App。

## 一、产品定位

这是一个面向游客和女生的陪伴型拍照搭子平台，不是传统专业摄影平台，也不是商业大片拍摄平台。

平台帮助普通用户找到一个懂审美、会拍照、能陪伴、能聊天、理解社交媒体出片需求的陪拍搭子。用户可以和陪拍者一起逛街、吃饭、探店、Citywalk、旅行、看展，并在过程中自然拍出适合发朋友圈、小红书、Instagram 的照片。

平台核心不是严肃的摄影服务，而是：
- 情绪价值
- 陪伴感
- 会沟通
- 懂女生需求
- 懂社交媒体出片
- 轻松自然，不尴尬
- 像朋友一样陪用户完成一次好看的出行体验

平台整体体验结合 Instagram 和 Airbnb：
- 首页浏览体验像 Instagram，以高质量图片流为核心。
- 服务预约体验像 Airbnb，根据地点、时间、活动形式、服务范围和价格匹配合适陪拍者。

平台长期方向不是单纯的陪拍交易工具，而是“陪拍内容社区 + 本地搭子社交 + 平台担保交易”的综合平台。用户既可以在平台内发现好看的作品和真实出行动态，也可以关注喜欢的陪拍者、参与话题圈子、评论收藏内容，并最终在平台内完成安全交易。

平台必须把社交和交易连接起来：社交负责建立信任，内容负责持续留存，信用和保障负责让用户愿意留在平台内交易，而不是私下跳单。

## 二、核心目标

1. 用吸引人的图片流增加用户停留时间。
2. 让用户通过图片发现喜欢的拍摄风格和生活场景。
3. 根据用户选择的地点、时间、活动形式和需求，匹配合适陪拍者。
4. 让用户在被图片打动后，快速、轻松、安全地完成预约。
5. 帮助陪拍者展示作品、设置服务范围、管理时间、定价和接单。
6. 通过严格审核、平台担保、消息风控和举报机制保障女生用户安全。
7. 防止跳单、私下交易和不安全线下见面。
8. 建立用户、陪拍者之间的关注关系，让一次约拍沉淀为长期信任。
9. 通过动态、评论、收藏、关注、话题和圈子提升社区留存。
10. 让平台内信用、评价、等级、曝光、担保和售后保障成为防跳单的核心动力。

## 三、用户端核心流程

用户端需要支持两条自然路径：内容社交路径和交易预约路径。

内容社交路径：
用户打开 App
→ 左上角选择城市、定位或搜索具体地址
→ 浏览类似 Instagram 的图片流和真实出行动态
→ 用户被图片、地点、风格或生活方式吸引
→ 点击进入帖子详情
→ 点赞、收藏、评论、关注作者
→ 进入作者主页查看更多作品、动态、评价和可预约信息
→ 持续关注、参与话题或圈子

交易预约路径：
用户打开 App
→ 搜索城市、地点、商圈、景点、餐厅或咖啡店
→ 浏览命中地点和时间的作品/动态
→ 点击帖子详情或作者主页
→ 点击陪拍者入口
→ 弹出预约面板
→ 查看陪拍者真人照片、安全标识、性格标签、可选地点、时间、活动形式、时长和价格
→ 点击“约TA陪拍”
→ 下单支付，平台托管款项
→ 进入订单绑定聊天
→ 完成服务
→ 评价并沉淀信用

产品原则：
允许用户在平台内自然认识、互动和建立信任，但涉及具体交易、联系方式、收款和线下见面时，必须引导用户进入平台内订单和担保交易流程。

## 四、首页图片流

首页采用类似 Instagram 的图片内容流，以陪拍者作品、真实客片、用户晒图、城市打卡照、探店照、旅行照、风格样片和出行动态为核心内容。

首页重点是让用户被图片本身吸引，形成持续浏览体验。

首页每组图片只展示极少量信息：
- 拍摄地点，例如“上海 · 武康路”
- 拍摄时间，例如“傍晚 / 夜景 / 春季 / 2026年5月”
- 作者头像或轻量身份标识
- 点赞、收藏、评论等轻量社交反馈

首页不直接展示陪拍者姓名、价格、服务内容、长文字介绍或明显预约卡片，避免破坏图片流沉浸感。

首页左上角提供城市/地点搜索入口。用户选择地点后，首页图片流优先展示该地点附近、服务范围命中、时间可约的陪拍者作品，也可以混合展示该地点相关的用户动态、热门路线、探店内容和话题内容。

首页可以包含轻量频道：
- 推荐
- 关注
- 附近
- 热门地点
- 圈子/话题

“推荐”优先展示高质量图片和高转化内容；“关注”展示用户关注的人发布的作品和动态；“附近”展示当前城市和地点附近的陪拍内容；“圈子/话题”承接社区讨论和经验分享。

产品原则：
首页不急着卖服务，而是先制造“我也想拥有这种照片”和“我想认识这个风格的人”的冲动。

## 五、帖子详情页

用户点击首页某组图片后进入帖子详情页。

帖子详情页仍然以图片为主，不做复杂服务页。

页面展示：
- 当前图片组大图浏览
- 拍摄地点
- 拍摄时间
- 帖子文字说明
- 作者头像、昵称和认证标识
- 点赞
- 收藏
- 评论
- 关注作者
- 相关话题
- 底部陪拍者入口

底部陪拍者入口可以展示：
- 陪拍者头像
- 昵称
- 一个轻量按钮，例如“约TA陪拍”
- 安全标识，例如“已实名认证”“平台担保”

如果用户想了解陪拍者更多历史作品，可以在帖子详情页向右滑动，查看该陪拍者过往发布的历史帖子图片。

评论区用于轻量交流、提问和真实反馈，但不能成为绕过平台交易的联系方式交换区。评论区同样需要敏感信息识别、举报入口和人工审核能力。

## 六、个人主页、关注与圈子

平台需要为普通用户和陪拍者都提供个人主页。个人主页不是复杂资料页，而是内容、信用和交易能力的集中展示。

普通用户主页展示：
- 头像
- 昵称
- 简介
- 城市
- 发布过的动态
- 收藏公开集合
- 关注和粉丝
- 评价记录
- 平台信用状态

陪拍者主页展示：
- 真人头像和生活照
- 自我介绍视频入口
- 作品动态
- 服务城市和可服务区域
- 擅长风格和互动方式
- 认证标识
- 评分、成交量、复购率、响应速度
- 用户评价
- 可预约入口

关注关系：
- 用户可以关注陪拍者、普通用户、话题和圈子。
- 关注后可在“关注”信息流中看到对方新作品、新动态和可约时间。
- 关注关系用于提升留存和信任，但不能自动放开联系方式交换。

圈子和话题：
- 平台后期需要支持城市圈、地点圈、风格圈、活动圈和新手经验圈。
- 示例：上海 Citywalk、东京旅行拍照、第一次约拍、女生安全约拍、探店拍照姿势、夜景出片。
- 圈子内容可以包括图文动态、经验帖、提问、避坑、路线推荐和陪拍者作品。
- 圈子需要管理员、内容审核、举报处理和热门话题运营。

社交产品原则：
平台允许用户自然交流、建立关系和形成社区氛围，但所有与具体订单、收款、联系方式和线下见面相关的行为，都应该被引导到平台内预约、订单聊天、担保支付和售后保障体系中。

## 七、预约弹窗

用户点击帖子详情页底部陪拍者入口后，不跳转复杂页面，而是直接弹出简洁预约面板。

预约面板顶部需要展示陪拍者真人个人照片，最好是自然、亲切、有笑容、有感染力的照片。

原因：
陪拍服务不只是买照片，更是买一次陪伴、沟通和情绪价值。真人照片可以增强信任感、亲近感和安全感，尤其适合游客和女生用户。

预约面板展示：
- 陪拍者真人照片
- 昵称
- 简短个人标签，例如“会指导动作”“轻松聊天”“适合第一次拍照”“懂女生拍照需求”
- 安全标识，例如“已实名认证”“视频已审核”“作品已审核”“平台担保”
- 可预约地点
- 可预约日期和时间段
- 可选择活动形式
- 可选择陪伴/拍摄时长
- 基础价格
- 附加服务价格，例如精修每张多少钱、加急出图多少钱、短视频拍摄多少钱
- 成交按钮：“约TA陪拍”

预约面板要简洁、美观、好上手，避免复杂表单感。选择项可以使用轻量 chips 或分组按钮。

## 八、收费方式

平台收费方式以活动形式和陪伴时长为主，而不是传统摄影棚套餐。

陪拍者可以设置：
- 活动类型：逛街拍照、吃饭探店、Citywalk、旅行跟拍、看展拍照、夜景散步、情侣/闺蜜陪拍等
- 陪伴时长：1小时、1.5小时、2小时、半天等
- 基础价格：按活动类型和时长收费
- 附加服务：精修每张加多少钱、加急出图多少钱、短视频拍摄多少钱等

示例：
- Citywalk 陪拍｜2小时｜399元
- 探店吃饭陪拍｜1.5小时｜299元
- 夜景散步陪拍｜1小时｜199元
- 旅行半日陪拍｜4小时｜699元
- 精修加购｜30元/张

## 九、用户端搜索与筛选

用户可以在首页左上角选择城市、当前定位或搜索具体约拍地址。

搜索支持：
- 城市
- 商圈
- 街道
- 景点
- 餐厅
- 咖啡店
- 展览馆
- 当前定位附近

用户还可以轻量筛选：
- 约拍日期
- 约拍时间段
- 活动类型
- 预算范围
- 想要的出片风格
- 是否需要会指导动作
- 是否适合第一次拍照
- 是否需要同性陪拍
- 只看女陪拍者
- 同性陪拍优先

筛选不应该破坏图片流美感，要轻量呈现。

## 十、陪拍者端功能

陪拍者端需要像轻量创作者工具，而不是复杂后台系统。

### 1. 入驻审核

陪拍者必须经过严格审核后才能接单。

需要提交：
- 手机号验证
- 真实姓名
- 身份证/护照等证件认证
- 人脸识别
- base 城市
- 紧急联系人
- 真人头像
- 生活照
- 自我介绍视频
- 作品样片
- 平台规则确认

自我介绍视频用于确认陪拍者为真人，判断沟通表达能力、亲和力和情绪价值，降低用户对线下见面的不安，防止虚假账号和盗图。

### 2. 资料编辑

陪拍者可以设置：
- 昵称
- 真人照片
- 性别
- 年龄段
- 简短介绍
- 性格标签
- 擅长风格
- 擅长互动方式，例如会聊天、会指导动作、安静陪伴、适合第一次拍照
- 是否展示自我介绍视频

### 3. 服务范围设置

陪拍者首先设置 base 城市，也就是主要接单城市。

然后设置可接受服务场景：
- 户外街拍
- Citywalk
- 商圈逛街
- 餐厅探店
- 咖啡店
- 展览/美术馆
- 夜景散步
- 景点旅行
- 室内空间
- 仅限商圈和餐厅
- 不接受户外
- 不接受夜间
- 不接受偏远地点

陪拍者可以在城市地图上勾选可服务区域。可服务区域可以包括：
- 行政区
- 商圈
- 街区
- 景点
- 地铁站附近
- 自定义半径范围

MVP 第一版可以先用城市 + 商圈/地点多选，后续升级为地图圈选。

### 4. 时间和价格设置

陪拍者设置：
- 可预约日期
- 可预约时间段
- 临时接单开关
- 活动形式
- 时长
- 基础价格
- 精修加购价格
- 加急出图价格
- 短视频价格

### 5. 作品发布

陪拍者像发 Instagram 帖子一样发布作品。

发布内容：
- 上传一组图片
- 填写拍摄地点
- 填写拍摄时间
- 帖子文字
- 风格标签
- 活动类型
- 是否展示到首页

作品需要经过平台审核后才进入首页图片流。

### 6. 订单管理

陪拍者可以查看：
- 待确认订单
- 已确认订单
- 今日行程
- 已完成订单
- 已取消订单
- 与用户沟通
- 确认完成

### 7. 收入页面

展示：
- 已完成订单收入
- 待结算金额
- 可提现金额
- 提现记录
- 平台服务费

## 十一、匹配逻辑

当用户搜索一个具体地址、街道、商圈、景点、餐厅或咖啡店时，系统需要：

1. 将用户输入的地址解析成城市、经纬度和地点类型。
2. 查找 base 城市与用户城市一致的陪拍者。
3. 判断用户地址是否落在陪拍者 ServiceArea 服务范围内。
4. 判断用户需求场景是否符合陪拍者接受的服务场景，且不在拒绝场景中。
5. 判断用户选择的日期和时间是否落在陪拍者可预约时间内。
6. 判断用户需要的时长是否有对应活动价格。
7. 如果用户有预算、风格、性别、安全偏好，则继续过滤。
8. 返回匹配陪拍者的作品帖子，并按图片质量、地点相关性、时间匹配度、评分、近期活跃度排序。

核心闭环：
图片吸引 → 社交互动 → 关注沉淀 → 地点命中 → 时间命中 → 陪拍者可服务 → 用户挑选 → 平台内成交 → 评价信用继续提升曝光

## 十二、消息页与防跳单机制

消息系统需要同时支持社交关系和交易安全，不能简单做成完全封闭，也不能完全开放成无边界私聊。

消息体系分为两层：

1. 社交私信
用于关注后轻量交流、内容反馈、普通提问和建立信任。社交私信可以存在，但需要控制交易风险。

社交私信能力：
- 文字
- 表情
- 图片
- 帖子分享
- 主页分享
- 平台安全提醒
- 举报入口
- 风险提示

社交私信限制：
- 未下单前不允许交换联系方式。
- 未下单前不允许发送外部支付方式、二维码、银行卡、微信、手机号等内容。
- 未下单前不鼓励约定具体线下见面细节，应引导进入预约流程。
- 高频私信、批量引流、疑似广告和诱导跳单需要触发风控。

2. 订单聊天
用于用户下单后围绕具体订单沟通，是平台保障交易的主沟通场景。

订单聊天能力：
- 地点确认
- 拍摄前需求沟通
- 穿搭和风格沟通
- 时间确认
- 平台内报价和加购
- 图片/参考图发送
- 订单状态提醒
- 平台安全提醒
- 举报入口
- 客服介入入口
- 服务完成确认
- 售后和纠纷证据留存

消息风控机制：
平台需要设置屏蔽词、敏感内容检测、行为风控和人工审核，防止跳单、私下交易和不安全见面。

需要识别：
- 微信、VX、V信、微、加我
- 电话号码
- QQ
- 小红书号
- Instagram、ins、ig
- Telegram、WhatsApp、Line
- 支付宝、银行卡、转账
- 私下付、线下付、绕平台
- 二维码、外部链接、收款码
- “平台太贵”“走外面”“私下便宜”“线下给你”等诱导语义
- 谐音、拆字、空格、符号规避写法

处理方式：
- 阻止发送
- 替换敏感内容
- 提示“为保障双方安全，请在平台内完成沟通和交易”
- 提供平台内替代动作，例如“发起预约”“修改订单”“申请加购”“联系客服”
- 多次违规后限制聊天
- 严重违规提交人工审核
- 可冻结订单或暂停陪拍者接单

防跳单的产品原则：
不要只依赖屏蔽词。平台要通过担保交易、售后仲裁、退款保障、评价信用、曝光权重、等级权益和保证金机制，让用户和陪拍者主动愿意留在平台内交易。

## 十三、订单、支付与平台规则

平台主要通过订单抽成盈利。用户按活动形式和陪伴时长支付费用，平台托管款项，服务完成后扣除平台服务费并结算给陪拍者。

平台抽成比例需要支持后台配置。

平台内交易需要提供私下交易无法获得的权益：
- 平台资金托管
- 订单凭证
- 售后仲裁
- 退款保障
- 纠纷处理
- 安全提醒
- 评价与信用累计
- 陪拍者等级成长
- 首页曝光权重
- 成交记录背书
- 用户复购和收藏沉淀

防跳单的核心不是只拦截联系方式，而是让平台内交易更安全、更有保障、更能积累信用和曝光。

订单状态：
- 待确认
- 已确认
- 已完成
- 已取消
- 退款中
- 争议处理中

支付规则：
- 用户下单时支付全款。
- 平台托管资金。
- 陪拍者确认后订单生效。
- 服务完成后用户确认或系统自动完成。
- 资金进入结算期。
- 若发生投诉或争议，资金暂缓结算。

取消规则：
- 陪拍者未确认前，用户可免费取消。
- 已确认后，根据距离服务开始时间判断退款比例。
- 临近开始时间取消可能扣除取消费。
- 陪拍者取消需要说明原因，多次取消会影响评分、曝光和接单权限。
- 平台应支持重新匹配其他陪拍者。

爽约规则：
- 用户未按时到达且超过宽限时间，可判定为用户爽约。
- 陪拍者无故迟到、失联或未到场，用户可申请退款。
- 平台可处罚陪拍者并重新匹配。

纠纷处理：
平台需要处理迟到、爽约、服务时间不足、地点不符、照片质量严重不符、精修延迟、态度差、不安全行为、骚扰和跳单等问题。

处理时参考：
- 订单信息
- 聊天记录
- 定位签到
- 图片交付记录
- 付款记录
- 评价
- 举报材料

违规和信用规则：
- 陪拍者引导跳单会影响曝光、接单权限、等级、保证金和账号状态。
- 用户多次诱导私下交易会影响私信权限、下单权限和账号信用。
- 平台外交易不累计评分、成交量、等级、曝光权重和平台保障。
- 平台内完成的订单才进入陪拍者主页信用展示。

## 十四、安全机制

因为平台主要面向游客和女生，安全感是核心转化因素。

平台需要：
- 严格审核陪拍者
- 实名认证
- 人脸识别
- 自我介绍视频审核
- 作品审核
- 平台担保
- 同性陪拍优先
- 只看女陪拍者
- 举报入口
- 客服介入
- 消息防跳单
- 平台内交易保障
- 主页信用展示
- 成交记录展示
- 评价真实性校验
- 高风险私信和评论拦截
- 线下见面安全提醒

平台不鼓励偏僻地点、深夜单独见面和私密空间服务。对夜间、偏远或高风险地点应增加安全提示或限制。

## 十八、视觉设计方向

视觉风格：
- 自然
- 干净
- 有呼吸感
- 亲近
- 真实
- 轻微精致感
- 社交媒体友好
- 有社区感
- 有信任感

首页要最大程度突出图片本身。不要做传统服务列表，不要在首页堆满价格、标签、按钮和销售话术。

社交互动控件要轻量，不要喧宾夺主。点赞、评论、收藏、关注、分享、进入主页等操作应该像 Instagram 一样自然存在，但不能把首页变成复杂运营活动页。

整体颜色建议：
- 白色或浅色背景
- 黑色/深灰文字
- 少量温暖强调色
- 安全标识可用柔和绿色或蓝色
- 主按钮可用黑色、珊瑚色、玫瑰色或暖红色

图片风格：
- 自然光
- 真实游客/女生出行
- Citywalk
- 探店

- 咖啡店
- 餐厅
- 夜景
- 看展

## 十九、整体设计原则

1. App 要尽可能简单、好上手、有美感。
2. 首页先用图片制造心动，而不是直接卖服务。
3. 平台不是传统摄影平台，而是陪伴型拍照搭子平台。
4. 用户买的不只是照片，还有陪伴感、理解、情绪价值和社交媒体出片。
5. 预约流程要短，不让用户进入太多页面。
6. 陪拍者真人照片和自我介绍视频用于建立信任、亲近感和安全感。
7. 收费按活动形式和时长，不按传统摄影棚套餐。
8. 地点、时间、服务范围匹配是促成交易的核心。
9. 消息系统必须防止跳单、私下交易和不安全约见。
10. 陪拍者审核要严格，因为平台涉及线下见面和女生用户安全。
11. 后台要能支撑审核、风控、举报、纠纷、结算和内容运营。
12. 平台要把“安全感”和“亲近感”同时做出来。
13. 平台要把社交关系、内容沉淀、信用体系和平台担保交易连成闭环。
14. 社交功能不是为了放任私下交易，而是为了在平台内建立信任并提高成交转化。
15. 防跳单不能只靠屏蔽词，要靠平台保障、信用沉淀、曝光权益、等级权益和违规成本共同完成。
16. 私信、评论、圈子和订单聊天都需要风控，但风控要尽量用平台内替代动作承接用户需求。
