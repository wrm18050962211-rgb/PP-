# PP App Store 上架路线

这份清单服务于当前最快路线：把 `pp-app` 现有 React/Vite Web App 先封装为 iOS App，进入 TestFlight，再冲 App Store 审核。

## 当前状态

- `pp-app` 已加入 Capacitor 配置，iOS 原生工程需要在 Mac 上生成。
- Web 构建产物目录是 `pp-app/dist`。
- App ID 暂定为 `com.ppplatform.app`，提交前可以改成公司或个人开发者账号下的正式 Bundle ID。
- 第一版目标是 App Store 最小可审版本，不追求完整平台自动化。
- 产品决策：支付必须进入第一版；摄影师资料审核、内容审核、交易纠纷和退款处理可以先人工运营。

## Mac 上第一步

在 Mac 上复制仓库后进入 `pp-app`：

```bash
npm install
npm run ios:add
npm run ios:open
```

如果已经生成过 `ios/` 工程，后续改前端后运行：

```bash
npm run ios:sync
npm run ios:open
```

## 第一版范围

保留：

- 首页作品流
- 摄影师/陪拍详情
- 咨询或预约入口
- 定金或订单支付
- 基础登录/注册入口
- 我的页面
- 订单支付状态页
- 客服/举报/隐私入口

暂缓：

- 自动摄影师资料审核
- 自动内容审核
- 自动交易纠纷仲裁
- 自动退款
- 自动结算和提现
- 完整双边平台冷启动机制
- 大规模推荐算法

## 第一版交易闭环

第一版采用“真实支付 + 人工运营”的窄闭环：

```text
用户浏览作品/摄影师
-> 提交预约
-> 支付定金或全款
-> 平台人工确认订单
-> 人工拉群/撮合/履约
-> 后台人工改状态
```

建议先做定金支付，不做完整托管结算。首版可只保留一个固定定金，例如 `99 元预约定金`，降低退款、纠纷和履约压力。

## 支付方案

陪拍属于线下真实服务，第一版可以按线下服务预约支付设计。需要在提交审核前确认 Apple 审核边界，并在 App Store Connect 审核说明里写清楚这是线下摄影/陪拍服务预约，不是数字内容购买。

国内第一版建议微信支付或支付宝二选一，优先选择当前最容易办通商户号和 App 支付配置的一方。

最小支付模块：

- 后端创建真实订单和支付单。
- 后端调用微信/支付宝预下单。
- 前端在 iOS App 内调用真实支付能力。
- 后端提供 HTTPS 支付回调地址。
- 后端验签并更新支付状态。
- 前端支付完成后查询服务端支付状态，不只相信客户端返回。
- 退款先走客服人工处理，后台人工标记状态。
- 结算和提现先线下或后台人工处理。

支付相关订单状态建议：

```text
pending_payment
deposit_paid
paid_pending_confirm
confirmed
in_service
completed
cancelled
refunding
refunded
disputed
```

## 支付上线前必须具备

- 商户号和 App 支付权限。
- 线上 HTTPS API 域名。
- 真实数据库写入订单、支付、支付渠道流水号。
- 支付回调验签。
- 支付结果查询接口。
- 支付失败、取消、超时的前端提示。
- 用户协议和支付说明。
- 客服入口和人工退款流程说明。

## App Store Connect 必备材料

- Apple Developer Program 账号
- App 名称、Bundle ID、SKU
- App 图标
- iPhone 截图
- App 分类、年龄分级
- 隐私政策 URL
- 用户协议 URL
- App Privacy 数据收集说明
- 审核账号和审核说明
- 支付/线下服务说明
- 联系方式

## 技术上线前必须补齐

- 生产 API 地址，不能使用 `127.0.0.1`。
- 真实账号体系，不能依赖本地验证码和 localStorage 登录。
- 真实图片/视频上传，不能只保存 data URL。
- PostgreSQL 或其他生产数据库的真实写入。
- 订单和支付状态持久化。
- 基础错误页、空状态和网络失败提示。
- 隐私政策与权限弹窗文案。
- iPhone 真机测试和 TestFlight 测试。

## 最短时间表

- 1 天：Mac 上生成 iOS 工程，跑通模拟器。
- 3-7 天：整理最小可审功能，上传 TestFlight。
- 1-2 周：接通真实支付链路、订单写入和回调验签。
- 2-4 周：补生产登录、存储、隐私和审核材料，提交 App Store 审核。
