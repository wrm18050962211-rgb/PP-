# PP陪拍平台 MVP

移动端优先的 Web App 初始化版本。

## 产品方向

PP 的核心闭环仍然是：发现作品 -> 咨询摄影师 -> 报价 -> 下单 -> 平台内沟通 -> 拍摄交付 -> 双方共同编辑作品 -> 可选择同步到主页。

后续产品会在“正式拍摄前”加入妆造能力，分两步演进：

1. 化妆：引入“化妆师”身份。化妆师和摄影师一样需要提交资料并经过管理员后台审核，审核通过后才能对外展示和接单。创作者在找摄影师之前，可以先按地点、时间、风格和预算筛选化妆师，完成化妆预约后再进入摄影师预约流程。
2. 穿衣：等 LT 项目可用后再接入。PP 根据拍摄地点沉淀该地点常见穿搭、风格和购买线索，用户可以把目标地点和风格带入 LT，在对应背景下试穿同类衣服，再决定是否购买或带到拍摄现场。

MVP 阶段不把化妆师和穿衣能力写进主流程代码，只在产品和数据设计中预留：

- 化妆师是独立服务身份，不等同摄影师。
- 化妆服务需要后台审核、服务范围、档期、报价、订单、聊天和风控。
- 摄影预约前可以串联化妆预约，但两个订单要保持独立，避免结算和纠纷混在一起。
- 穿衣能力先作为 LT 外部能力接入，PP 只负责提供地点、风格、作品参考和购买/试穿入口。

## 技术栈

- React + TypeScript
- Vite
- React Router
- Tailwind CSS
- lucide-react 图标

## 目录说明

- `src/features/user`：用户端页面：首页图片流、帖子详情、订单、消息、我的
- `src/features/companion`：陪拍者端工作台
- `src/features/admin`：运营后台 MVP
- `src/components`：跨页面复用组件
- `src/data`：MVP 阶段模拟数据
- `src/layouts`：应用外壳与底部导航

## 本地运行

```bash
npm install
npm run dev -- --host 127.0.0.1 --port 5174
```

常用查看地址：

- 用户端：http://127.0.0.1:5174/consumer
- 陪拍者端：http://127.0.0.1:5174/companion
- 运营后台：http://127.0.0.1:5174/admin
